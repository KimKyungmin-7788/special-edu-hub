# CLAUDE.md

이 파일은 작업할 때마다 반드시 지켜야 하는 규칙이다. PRD.md와 함께 읽는다.
PRD.md = 무엇을 만드는가. 이 파일 = 어떻게 작업하는가.

## 프로젝트

강원 특수교육 디지털 학습자료 누리집(가칭). 특수교사가 만든 교육 웹앱을 모아 보여주는 공개 카탈로그.
**현재 범위: 0~1단계 "작동하는 카탈로그"까지만.** 자세한 범위는 PRD.md 참고.

## 기술 스택 (변경 금지)

React + Vite / Tailwind CSS + shadcn/ui / Supabase(1단계부터) / 배포 Vercel(추후).
새 라이브러리·프레임워크를 임의로 추가하지 않는다. 필요하면 먼저 물어본다.

## 절대 규칙

1. **색·폰트·반경·여백은 CSS 변수 토큰으로만 사용한다. 하드코딩 금지.**
   컴포넌트 안에 `#333`, `bg-blue-500` 같은 직접 색을 박지 않는다.
   브랜드 미정 상태이므로 토큰만 바꾸면 전체가 바뀌어야 한다.

2. **디자인은 중립으로.** 화이트·블랙·그레이 + 기본 시스템 폰트.
   "담백하지만 완성도 있게." 화려한 색·그라데이션·장식 넣지 않는다.

3. **로고·누리집명·히어로 문구는 config 한 곳에서 불러온다.** 화면에 직접 쓰지 않는다.

4. **범위 밖 기능을 임의로 구현하지 않는다.**
   회원가입/로그인, 교사인증, 앱 등록, 좋아요·담기 누적/토글, 조회수 누적, 댓글·답글·멘션 = **이번 단계 금지.**
   이들은 UI 자리(껍데기)만 두거나 "준비 중"으로 표시. 동작은 만들지 않는다.

5. **화면 구조(레이아웃·정보 배치·페이지 흐름)는 제대로 잡는다.**
   나중에 바꾸는 건 표면 스타일이지 구조가 아니다.

## 작업 방식

- **한 번에 한 단계씩.** 0단계 완료·사용자 확인 후 1단계로 간다. 0과 1을 한꺼번에 만들지 않는다.
- **큰 변경(새 페이지, 스키마, 구조 변경, 라이브러리 추가) 전에는 먼저 물어본다.** 바로 코딩하지 않는다.
- 각 단계가 끝나면 사용자가 `npm run dev`로 직접 확인한다. 확인 후 Git 커밋.
- 한 번에 거대한 코드를 쏟지 말고, 작게 나눠 만들고 확인받는다.

## Supabase 규칙 (1단계)

- 스키마·RLS·시드 SQL은 **네가 작성하되, 직접 실행하지 않는다.**
- "이 SQL을 Supabase 대시보드에서 실행하세요" 형태로 사용자에게 넘긴다. 사용자가 직접 적용한다.
- 이번 단계 RLS는 공개 읽기(SELECT)만. 쓰기 정책은 후속 단계.
- 앱 데이터는 시드로 채운다(PRD 7번 참고).

## 앱 실행 방식

교육 웹앱은 **새 탭(링크)** 으로 연다. `target="_blank" rel="noopener"`. iframe 사용 금지.
